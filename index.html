Installing SSLfish

There are two basic methods to install the SSLfish package, from within R itself using the devtools package and downloading a binary appropriate to your operating system. If you are using a Windows machine it is possible that you will need Rtools installed to build and install the package. There is, however, no compiled code in SSLfish.

Using devtools

You can obtain the SSLfish package using the GitHub installation function within the devtools R package. If devtools is not installed, it can be installed from CRAN by typing install.packages("devtools") from within the R console.

If devtools is installed (or after you have installed it), to install SSLfish, open the R console and type

> library(devtools)
> install_github("SSLfish", username="NMML")

Downloading a binary file

If you are using a Windows or Mac OSX machine, you can also download prebuilt binary files for installation. The Mac OSX version is available here. The windows version is downloaded here. If you are running a Linux machine, see the following section. Note: make sure your browser does not try to automatically unpack the file (this is the default for Safari, go into preferences to change this feature).

After you download the file, open R and set the working directory (e.g., using setwd("path/to/download/dir/")). Once this is set, run

> install.packages("SSLfish_0.11.zip", repos=NULL)

If you have a mac replace with "SSLfish_0.11.zip" with "SSLfish_0.11.tgz".

Downloading the source file

Source files for each version can be downloaded here. The desired file can be downloaded and installed in the same way as with the binaries, e.g., 

> install.packages("SSLfish_0.11.tar.gz", repos=NULL, type="source")

Is it installed?

However you choose to install you should see some output that, hopefully, ends with:

* installing *source* package 'SSLfish' ...
** R
** data
** demo
** inst
** preparing package for lazy loading
** help
*** installing help indices
** building package indices
** testing if installed package can be loaded
* DONE (SSLfish)
>

In which case, everything is installed.

Using SSLfish

To begin using SSLfish simply open the R console and type 

> library(SSLfish)

Following this you should see this output:

Loading required package: gtools

 SSLfish 0.11 (2013-11-15) 

>

It will throw some errors if the required packages are not installed. Just install these separately using, e.g., install.packages("gtools"). From here you can look at the code for running simulations. Type ?SSLfish to get the help page index.
